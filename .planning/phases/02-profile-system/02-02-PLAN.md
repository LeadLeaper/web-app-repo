  <action>
Create JavaScript functions for rendering panel content following user's organization requirements:

1. **Main Rendering Function** (openProfilePanel):
   - Accepts contactData object with all profile information
   - Calls renderProfileContent to populate panel HTML
   - Applies smart default expansion rules
   - Does NOT persist section state (per user requirement - always use smart defaults)

2. **Content Structure Function** (renderProfileContent):
   - Renders fixed header: photo, name, LIVE badge, status
   - Renders fixed actions: prev/next arrows, email, call, more
   - Calls renderSections for scrollable content
   - Returns complete HTML string

3. **Section Organization** (renderSections):
   Define sections in user-specified order:

   **Group 1 - Static Information:**
   - Details (always expanded per user requirement)
   - Company Info (always expanded per user requirement)
   - Social/Professional Links (expand if has content)

   **Group 2 - Activity:**
   - Notes (expand if has content, else show "Add Note" prompt)
   - Calls (expand if has content, else show "Log Call" prompt)
   - Meetings (expand if has content, else show "Schedule Meeting" prompt)
   - Tasks/Reminders (expand if has content, else show "Add Reminder" prompt)

   **Group 3 - Engagement:**
   - Email Replies (expand if has content)
   - Email Links Viewed (expand if has content)
   - Emails Sent (expand if has content)

   Return array of section HTML strings

4. **Individual Section Renderer** (renderSection):
   - Accepts section config: {id, title, alwaysExpanded, data, group}
   - Determines expansion state using smart defaults:
     * alwaysExpanded === true → expanded class
     * alwaysExpanded === null && data.length > 0 → expanded class
     * Otherwise → collapsed
   - Generates section HTML with header (title + toggle icon) and content
   - Calls renderSectionContent for data items
   - Returns section HTML string

5. **Section Content Renderer** (renderSectionContent):
   - If no data: Return empty state HTML with actionable prompt
   - If has data: Map items to appropriate HTML based on section type
   - Use placeholder rendering for now (detailed content in plan 02-03)

6. **Section Toggle Handler** (initSectionToggles):
   - Bind click handler to .section-header elements
   - Toggle .expanded class on parent .profile-section
   - Use jQuery .slideToggle(300) for smooth animation (matches 300ms timing)
   - Called after content renders

Follow existing patterns from nav-left.js: jQuery event delegation, data-driven rendering, CSS class-based state management.

  <action>
Create CSS styling for collapsible profile sections following modern SaaS design patterns:

1. **Section Container** (.profile-section):
   - Border-bottom: 1px solid var(--color-neutral-200) (subtle divider)
   - Padding: var(--spacing-md) 0
   - Cursor: pointer on header

2. **Section Header** (.section-header):
   - Display: flex; justify-content: space-between; align-items: center
   - Font-weight: 600 (semi-bold)
   - Color: var(--color-neutral-700)
   - Padding: var(--spacing-sm) var(--spacing-md)
   - Hover background: var(--color-neutral-50)
   - Transition: background-color 200ms

3. **Section Toggle Icon** (.section-toggle):
   - Content: "▾" (down arrow)
   - Transform: rotate(-90deg) when parent NOT .expanded (points right)
   - Transform: rotate(0deg) when parent .expanded (points down)
   - Transition: transform 200ms ease
   - Color: var(--color-neutral-500)

4. **Section Content** (.section-content):
   - Padding: var(--spacing-sm) var(--spacing-md)
   - Display: none by default (jQuery slideToggle will manage)
   - When parent .expanded: display: block (managed by jQuery)

5. **Section Groups** (visual separation):
   - Add subtle spacing between groups (Static, Activity, Engagement)
   - Use .section-group wrapper with margin-top: var(--spacing-lg)
   - Or use :nth-child selectors to add spacing at group boundaries

6. **Empty State Styling** (.empty-state):
   - Text-align: center
   - Padding: var(--spacing-lg)
   - Color: var(--color-neutral-500)
   - Font-style: italic
   - Border: 2px dashed var(--color-neutral-200)
   - Border-radius: var(--radius-md)
   - Background: var(--color-neutral-50)

7. **Empty State Action Button** (.empty-state-action):
   - Display: inline-block
   - Margin-top: var(--spacing-sm)
   - Color: var(--color-primary)
   - Font-weight: 600
   - Cursor: pointer

   - Padding: var(--spacing-xs) 0
   - Border-bottom: 1px solid var(--color-neutral-100) (lighter divider)
